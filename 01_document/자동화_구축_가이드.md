## 0. 사전 준비 사항

자동화 구현을 시작하기 전에 아래의 구글 스프레드시트와 채널을 먼저 준비

### 1) 구글 스프레드시트 생성 (프로젝트 1용)
- 구글 드라이브에 **[자동화_과제_테스트]**라는 이름의 스프레드시트를 생성합니다.
- 첫 번째 행(헤더)에 아래 컬럼을 입력합니다.
  - `Name`, `Email`, `Score`, `Status`
- **테스트 데이터 입력** (나중에 트리거 테스트 시 사용):
  - 1행: `Hong`, `swartlab01@gmail.com`, `85`, `Pending` (합격 케이스)
  - 2행: `Lim`, `swartlab01@gmail.com`, `60`, `Pending` (불합격 케이스)
  - 3행: `Kim`, `swartlab01@gmail.com`, `95`, `Pending` (합격 케이스)
  - 4행: `Lee`, `swartlab01@gmail.com`, `72`, `Pending` (불합격 케이스)
  - 5행: `Park`, `swartlab01@gmail.com`, `50`, `Pending` (불합격 케이스)
  - 6행: `Choi`, `swartlab01@gmail.com`, `90`, `Pending` (합격 케이스)

### 2) 메신저 수신 채널 준비 (Discord)
- **Discord 사용 시**: 
  - 개인 테스트용 Discord 서버를 생성하거나 기존 서버 채널 설정에서 `연동(Integrations) > 웹훅(Webhooks) 생성`을 눌러 **웹훅 URL**을 복사해 둡니다.

## 1. [프로젝트 1] Make 구축 가이드

Make는 비주얼 드래그 앤 드롭 방식으로 무제한 다단계 분기(Router)가 무료 플랜에서도 가능합니다.

### 단계 1: 시나리오 생성 및 구글 시트 트리거 설정
1. [Make 사이트](https://www.make.com/)에 로그인하고 우측 상단의 **[Create a new scenario]**를 클릭합니다.
2. 화면 중앙의 `+` 버튼을 누르고 **Google Sheets**를 검색한 뒤 **[Watch Rows]**를 선택합니다.
3. **Connection**: `Add` 버튼을 눌러 본인의 구글 계정을 연동하고 로그인합니다.
4. **Spreadsheet**: 준비해 둔 `[자동화_과제_테스트]` 시트를 선택합니다.
5. **Sheet**: `시트1`을 선택합니다.
6. **Table contains headers**: `Yes`로 설정합니다.
7. **Limit**: `2`로 설정(한 번에 가져올 행 수)한 뒤 `OK`를 누릅니다.
8. `Choose where to start` 팝업이 뜨면 **[All]** 또는 **[Manually choose]**를 선택해 테스트 데이터를 선택합니다.

### 단계 2: Router(조건 분기) 추가
1. Google Sheets 모듈의 오른쪽에 마우스를 가져가면 나오는 연결선 끝의 `Add another module` 영역이나 하단의 **Flow Control** 메뉴에서 **Router**를 클릭해 연결합니다.
2. 라우터 뒤에 두 개의 경로(Branch)가 생성됩니다. (안 생기면 라우터를 더블 클릭하거나 드래그하여 경로 추가)

### 단계 3: 조건 분기(필터) 설정
1. **합격 경로 필터 설정**:
   - 라우터와 위쪽 경로 모듈 사이의 **점선(연결 링크) 중간에 있는 렌치 모양 아이콘**을 클릭하고 **[Set up a filter]**를 선택합니다.
   - **Label**: `80점 이상 (합격)`
   - **Condition**: Google Sheets에서 가져온 `Score` 필드를 선택합니다.
   - **Operator**: `Numeric operators: Greater than or equal to (>=)`를 선택합니다.
   - **Value**: `80`을 입력하고 `OK`를 누릅니다.
2. **불합격 경로 필터 설정**:
   - 아래쪽 경로의 연결 링크 점선을 클릭하고 **[Set up a filter]**를 선택합니다.
   - **Label**: `80점 미만 (불합격)`
   - **Condition**: `Score` 필드 선택.
   - **Operator**: `Numeric operators: Less than (<)` 선택.
   - **Value**: `80` 입력 후 `OK`.

### 단계 4: 액션 모듈 추가 (Gmail & Discord/Slack)
각 경로의 끝에 이메일과 메신저 알림을 보낼 액션을 연결합니다.

1. **80점 이상(위쪽) 경로**:
   - 끝에 모듈을 추가하고 **Gmail > Send an email** 선택.
   - **To**: 구글 시트의 `Email` 필드 매핑.
   - **Subject**: `[축하합니다] 합격 안내 메일`
   - **Content**: `안녕하세요 {{Name}}님, 테스트 결과 {{Score}}점으로 최종 합격하셨습니다!` 작성 후 `OK`.
   - Gmail 모듈 뒤에 하나 더 연결하여 **Discord** 추가. 
     - *Discord Webhook 사용 시*: **Discord > Post a Message (with Webhook)** 선택 후 사전 준비한 웹훅 URL 입력, Message 내용에 `📢 합격 알림: {{Name}}님이 {{Score}}점으로 합격하셨습니다.` 입력.
2. **80점 미만(아래쪽) 경로**:
   - 끝에 **Gmail > Send an email** 추가.
   - **To**: `Email` 매핑.
   - **Subject**: `[안내] 테스트 결과 안내 메일`
   - **Content**: `안녕하세요 {{Name}}님, 테스트 결과 {{Score}}점으로 불합격하셨습니다. 다음 기회에 도전해 주세요.`
   - 뒤에 **Discord (혹은 Slack)** 추가하여 `📢 불합격 알림: {{Name}}님이 {{Score}}점으로 기준 점수에 미달하였습니다.` 설정.

### 단계 5: 테스트 실행 및 화면 캡처
1. 좌측 하단의 **[Run once]**를 클릭합니다.
2. 구글 시트에 작성해 둔 데이터 2건이 감지되면서, 85점 데이터는 위쪽 경로로 흐르고, 60점 데이터는 아래쪽 경로로 흐르는 것이 **초록색 숫자 동그라미(성공 표시)**로 나타나는지 확인합니다.
3. **이 시점에서 Make 시나리오 캔버스를 캡처**하여 `02_img/proj1_make_workflow.png`로 저장합니다.
4. 본인의 Gmail과 Discord/Slack 채널에 실제로 발송된 합격/불합격 메시지를 캡처하여 `02_img/proj1_make_result.png`로 저장합니다.

## 2. [프로젝트 1] Zapier 구축 가이드

Zapier는 직관적이지만, 무료 플랜에서는 다단계(Multi-step) 또는 Paths(분기) 기능이 유료입니다. 이를 해결하기 위해 무료 플랜에서 실습할 수 있도록 **"단일 Zap에 하나의 필터를 걸어 합격자만 필터링하는 구조"**로 구현을 권장합니다.

### 단계 1: Zap 생성 및 트리거 설정
1. [Zapier 사이트](https://zapier.com/) 로그인 후 **[Create Zap]**을 클릭합니다.
2. **Trigger**: `Google Sheets`를 선택합니다.
3. **Event**: `New Spreadsheet Row` 선택 후 `Continue`.
4. **Account**: 본인의 구글 계정 선택 후 `Continue`.
5. **Trigger 설정**:
   - **Spreadsheet**: `[자동화_과제_테스트]` 시트 선택.
   - **Worksheet**: `시트1` 선택 후 `Continue`.
6. **Test trigger**를 눌러 구글 시트에 있는 테스트 행 데이터를 정상적으로 불러오는지 확인하고 `Continue with selected record`를 누릅니다.

### 단계 2: Filter(조건 분기) 추가
1. 트리거 바로 밑의 `+` 버튼을 눌러 **Filter by Zapier**를 추가합니다.
2. **Only continue if...** 설정:
   - 첫 번째 칸: 구글 시트의 `점수 (Score)` 필드를 선택합니다.
   - 두 번째 칸 (Condition): `(Numeric) Greater than` 또는 `Greater than or equal to` 선택.
   - 세 번째 칸: `80`을 입력하고 `Continue`를 누릅니다.
3. 테스트 실행 결과 "Your Zap would have continued..."(80점 이상 데이터의 경우)가 뜨는지 확인합니다.

### 단계 3: 액션 모듈 추가 (Gmail & Slack/Discord)
1. 필터 밑의 `+` 버튼을 눌러 **Gmail**을 추가합니다. (무료 플랜 14일 트라이얼 기간에는 멀티 액션이 허용되며, 트라이얼이 끝나면 단일 액션으로 축소될 수 있습니다.)
2. **Event**: `Send Email` 선택 후 계정 연동.
3. **To**: 구글 시트의 `Email` 필드 매핑.
4. **Subject**: `[축하합니다] Zapier 합격 안내`
5. **Body**: `안녕하세요 {{Name}}님, {{Score}}점으로 합격하셨습니다.` 입력 후 `Continue`.
6. `+` 버튼을 다시 눌러 **Slack** 또는 **Discord** 액션을 추가하고 동일하게 알림 메시지 구조를 작성합니다.
   - 예: Slack - `Send Channel Message`로 채널을 지정하고 `[Zapier 알림] {{Name}}님 합격!` 발송 설정.

### 단계 4: 테스트 실행 및 화면 캡처
1. Zap 우측 상단의 **[Publish]** 또는 **[Test step]**을 차례로 실행합니다.
2. **Zapier 워크플로우 편집 화면을 캡처**하여 `02_img/proj1_zapier_workflow.png`로 저장합니다.
3. 실제 실행된 이메일 또는 슬랙 화면을 캡처하여 `02_img/proj1_zapier_result.png`로 저장합니다.

## 3. [프로젝트 2] 자유 주제 (AI 연동 + 에러 처리) 가이드

- **플랫폼**: **Make** (무료로 Google AI Studio(Gemini) 모듈 연동 및 정교한 에러 핸들러 설정 가능)
- **목표**: 수신되는 고객 이메일을 AI가 분석하여 분류하고, 분류 기준에 따라 메신저로 즉시 알림을 보내거나 구글 시트에 백업합니다. 오류 발생 시 대체 경로(에러 핸들러)가 작동하도록 합니다.

### 단계 1: Google AI Studio API Key 발급
1. [Google AI Studio](https://aistudio.google.com/)에 로그인하여 좌측 메뉴의 **[API keys]**에서 API 키를 생성하고 복사해 둡니다. (Gemini API는 무료 티어를 제공하므로 추가 결제 없이 사용 가능합니다.)

### 단계 2: Gmail 트리거 설정
1. Make에서 새 시나리오를 만들고 **Gmail > Watch Emails** 모듈을 생성합니다.
2. **Folder**: `Inbox` (받은 편지함)
3. **Filter**: `Unread Only`를 `Yes`로 하거나, 특정 라벨 또는 검색어(예: `subject:문의`)를 걸어 필요한 메일만 읽어오도록 설정합니다.

### 단계 3: Google AI Studio (Gemini) 연동
1. Gmail 모듈 뒤에 **Google AI Studio (Gemini) > Create a Completion** 모듈을 추가합니다. (Make에 기본 내장된 모듈이므로 별도 설치 없이 검색하여 바로 사용 가능)
2. **Connection**: `Add` 버튼을 누르고 Google AI Studio API Key를 입력하여 연동합니다.
3. **Model**: `Gemini 2.0 Flash` 선택. (무료 티어에서 충분한 성능 제공)
4. **Messages / Prompt**:
   - **System Prompt**: `You are a customer service assistant. You must analyze the incoming email and return a JSON structure with three keys: "summary" (one-sentence Korean summary), "sentiment" (angry, negative, positive, or neutral), and "urgency" (high, medium, or low). Output ONLY raw JSON. No markdown formatting like \`\`\`json.`
   - **User Prompt**: `Subject: {{Subject}}, Body: {{Text Content}}` (Gmail에서 가져온 제목과 본문을 매핑)

### 단계 4: JSON 파서 추가 (AI 결과 파싱)
AI가 텍스트 형태로 뱉어낸 JSON 데이터를 Make에서 개별 필드로 사용하기 위해 JSON 파서를 씁니다.
1. Gemini 모듈 뒤에 **JSON > Parse JSON** 모듈을 추가합니다.
2. **JSON String**: Gemini 모듈의 결과값(`Text` 또는 `Content` 필드)을 매핑합니다.

### 단계 5: Router를 통한 긴급도 기준 분기 및 액션
1. Parse JSON 모듈 뒤에 **Router**를 연결합니다.
2. **긴급(Urgency가 'high' 또는 Sentiment가 'angry'인 경우) 경로**:
   - 필터 조건: JSON 파서에서 나온 `urgency` 필드가 `Equal to (case insensitive)` -> `high` 이거나 `sentiment` 필드가 `Equal to` -> `angry`.
   - **Action 1**: **Discord > Post a Message (with Webhook)**를 통해 담당자 멘션을 포함한 긴급 알림 전송.
     - 예: `🚨 긴급 문의 접수! AI 감정: {{sentiment}} | 요약: {{summary}}`
   - **Action 2**: **Google Sheets > Add a Row**로 긴급 관리 시트에 기록.
3. **일반 경로**:
   - 필터 조건: 위 조건을 만족하지 않는 모든 경우.
   - **Action**: **Google Sheets > Add a Row**로 일반 문의 아카이브 시트에 이메일 작성자, 제목, 요약, 접수 시각을 차례로 기록.

### 단계 6: 에러 핸들러(보너스 과제 2) 추가
API 호출 횟수 초과, 레이트 리미트 등으로 자동화가 멈추는 것을 방지합니다.
1. **Google AI Studio (Gemini) 모듈**을 마우스 우클릭하고 **[Add error handler]**를 선택합니다.
2. 제공되는 핸들러 옵션(Flow Control) 중 **[Ignore]** 또는 **[Resume]**을 추가합니다.
   - *Ignore* 적용 시: 에러가 발생해도 시나리오가 멈추지 않고 통과됩니다.
   - *Resume* 적용 시: 오류 시 대체할 디폴트 값(예: `summary: "분석 오류", urgency: "medium"`)을 지정해 다음 모듈로 넘겨줄 수 있습니다.
3. 에러 발생 시 알림 전송: 에러 핸들러 노드 끝에 **Gmail > Send an email**을 연결하여 관리자 이메일 주소로 `[자동화 에러] Gemini API 호출 중 오류가 발생하여 대체 값으로 시나리오를 계속 진행했습니다.`라는 경보 메일을 보냅니다.

### 단계 7: 테스트 실행 및 화면 캡처
1. 본인 메일 계정으로 제목에 `[문의] 환불해주세요`와 같이 화난 어조의 본문을 적어 이메일을 발송합니다.
2. Make 시나리오를 **[Run once]** 하여 메일 수신 -> Gemini AI 분석 -> 긴급 판정 및 Discord 발송이 잘 되는지 확인합니다.
3. **Make 시나리오 전체 맵(에러 핸들러 노드가 포함된 화면)을 캡처**하여 `02_img/proj2_make_workflow.png`로 저장합니다.
4. 메신저로 긴급 알림이 온 내역 및 구글 시트에 정상 아카이빙된 화면을 캡처하여 `02_img/proj2_make_result.png`로 저장합니다.
